# Neovim Theme Integration Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Integrate Neovim with the dotfiles theme engine so that `dotfiles theme apply <name>` also sets the Neovim colorscheme.

**Architecture:** Add Neovim colorscheme plugins matching the theme names in `system/themes/list/`. Create a new renderer `system/themes/renderers/nvim.sh` that generates a Lua file (`system/themes/generated/nvim-theme.lua`) setting `vim.g.theme_colorscheme` and `vim.g.theme_style`. Neovim loads this file **before** lazy.nvim initializes, so plugin specs read the globals to configure the correct variant.

**Tech Stack:** Lua (Neovim config), Bash (renderer), lazy.nvim (plugin manager)

**Startup flow:**
1. `init.lua` loads `nvim-theme.lua` (generated) → sets `vim.g.theme_colorscheme` + `vim.g.theme_style`
2. `require("config.lazy")` triggers lazy.nvim → plugin specs read `vim.g.theme_*` in `opts` functions
3. Plugin setup applies colorscheme with correct style

### Task 1: Create the Neovim theme renderer

**Files:**
- Create: `system/themes/renderers/nvim.sh`

- [ ] **Step 1: Create the renderer script**

```bash
#!/usr/bin/env bash
# Renderer: Neovim — generates nvim-theme.lua with colorscheme + style mapping.
set -euo pipefail

OUTPUT="${DOTFILES_DIR}/system/themes/generated/nvim-theme.lua"

# Map THEME_NAME → Neovim colorscheme name
declare -A COLORSCHEME_MAP=(
  [catppuccin-mocha]="catppuccin"
  [catppuccin-macchiato]="catppuccin"
  [dracula]="dracula"
  [everforest]="everforest"
  [gruvbox-dark]="gruvbox"
  [kanagawa]="kanagawa"
  [nord]="nord"
  [retro-phosphor]="gruvbox"
  [rose-pine]="rose-pine"
  [solarized-dark]="solarized"
  [tokyo-night]="tokyonight"
)

# Map THEME_NAME → style/variant (only for plugins that need it)
declare -A STYLE_MAP=(
  [catppuccin-mocha]="mocha"
  [catppuccin-macchiato]="macchiato"
  [gruvbox-dark]="dark"
  [kanagawa]="wave"
  [rose-pine]="main"
  [tokyo-night]="night"
)

name="${THEME_NAME:-gruvbox-dark}"
scheme="${COLORSCHEME_MAP[$name]:-$name}"
style="${STYLE_MAP[$name]:-}"

{
  echo "-- Generated by theme system. Do not edit by hand."
  echo "-- Set by: dotfiles theme apply ${name}"
  echo "vim.g.theme_colorscheme = \"${scheme}\""
  if [[ -n "$style" ]]; then
    echo "vim.g.theme_style = \"${style}\""
  fi
} > "$OUTPUT"

printf "  ✨ nvim-theme.lua\n"
```

### Task 3: Load generated theme and apply colorscheme at startup

**Files:**
- Modify: `core/nvim/init.lua`

- [ ] **Step 1: Add theme loading to init.lua**

Insert **before** `require("config.lazy")` to set globals, and **after** to apply colorscheme:

```lua
-- Load generated theme globals (set by `dotfiles theme apply`)
do
  local dotfiles = vim.env.DOTFILES_DIR or (vim.env.HOME .. "/dotfiles")
  local theme_file = dotfiles .. "/system/themes/generated/nvim-theme.lua"
  local f = loadfile(theme_file)
  if f then f() end
end

require("config.lazy")

-- Apply colorscheme from theme engine (after plugins are loaded)
if vim.g.theme_colorscheme then
  vim.cmd.colorscheme(vim.g.theme_colorscheme)
end
```

This replaces the existing `require("config.lazy")` line.
