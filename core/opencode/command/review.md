description: Code review with best practices and security focus
---

Perform a thorough code review of the current changes or specified files:

1. Read the code carefully
2. Check for:
   - Security issues (SQL injection, XSS, path traversal, etc.)
   - Performance bottlenecks
   - Maintainability concerns
   - Error handling gaps
   - Test coverage
3. Provide constructive feedback with specific line references
4. Suggest improvements with code examples where helpful

Be thorough but constructive. Praise good patterns as well as flagging issues.
